## Authentication

All protected endpoints require a Bearer JWT in the `Authorization` header:

```
Authorization: Bearer <access_token>
```

WebSocket feeds accept the token as a query parameter instead (browsers do not support custom
headers on WebSocket upgrades):

```
ws://localhost:8000/ws/market?token=<access_token>
```

Access tokens expire after **15 minutes** (configurable via `JWT_ACCESS_TOKEN_EXPIRE_MINUTES`).
Refresh tokens expire after **7 days** and are rotated on every use (token family rotation).

## Selected Endpoint Details

### Auth

| Method | Path | Auth | Description |
|--------|------|:----:|-------------|
| `POST` | `/api/v1/auth/login` | No | Returns `access_token` + `refresh_token` |
| `POST` | `/api/v1/auth/refresh` | No | Rotates refresh token; returns new pair |
| `POST` | `/api/v1/auth/logout` | Yes | Invalidates all refresh tokens for the user |
| `GET` | `/api/v1/auth/me` | Yes | Returns `{id, email, role, totp_enabled}` |
| `POST` | `/api/v1/auth/totp/setup` | Yes | Returns TOTP `secret` and `otpauth://` URI |
| `POST` | `/api/v1/auth/totp/verify` | Yes | Activates 2FA; persists secret to user record |

## Error Response Format

All errors follow FastAPI's default format:

```json
{
  "detail": "Human-readable error message"
}
```

Common status codes:

| Code | Meaning |
|------|---------|
| `401` | Missing or invalid JWT |
| `403` | Authenticated but insufficient role/permission |
| `404` | Resource not found |
| `422` | Request body validation error (Pydantic) |
| `429` | Rate limit exceeded (Redis sliding-window counter) |
| `502` | Upstream market data provider error |
