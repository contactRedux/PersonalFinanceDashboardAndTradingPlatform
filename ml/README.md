# ml/ — Machine Learning Models

## What is this folder?

This folder contains the platform's "crystal ball" — four machine learning models that analyze price data, market conditions, and news to make predictions or classifications.

**Machine learning** is the field of teaching computers to find patterns in data without being explicitly programmed with rules. Instead of writing "IF RSI < 30 AND volume is high, THEN buy," you show the model thousands of historical examples and let it figure out the patterns itself.

## The Four Models

### LSTM Price Predictor (`models/lstm/`)
**What it does:** Looks at the last 30 days of price data (including technical indicators) and predicts whether the next day's return will be **up (>+1%)**, **flat (±1%)**, or **down (<-1%)**.

**LSTM** (Long Short-Term Memory) is a type of recurrent neural network specifically designed for sequences — think of it like a human reading text: it remembers relevant context from earlier while processing the current word. For price data, "earlier context" means earlier price bars.

The model has:
- A 2-layer LSTM encoder with configurable hidden size (default 64)
- A linear classification head with 3 outputs
- Trained using PyTorch with cross-entropy loss

**Training:** `POST /api/v1/ml/lstm/train` → dispatches a Celery background job → calls `ml/models/lstm/train.py` → saves model weights to `data/ml_weights/lstm/{TICKER}.pt`

**Prediction:** `GET /api/v1/ml/lstm/predict?ticker=AAPL` → loads saved weights → returns `{up: 0.42, flat: 0.31, down: 0.27}`

### XGBoost Signal Classifier (`models/xgboost/`)
**What it does:** Outputs a binary **long (1) / no-position (0)** signal plus a confidence probability. Also provides feature importance — *which indicators contributed most to the current prediction?*

**XGBoost** (Extreme Gradient Boosting) is an ensemble of decision trees trained sequentially, each one correcting the errors of the previous. It's well-suited for tabular data with many features.

**Features used:** OHLCV relative values, SMA-10/20/50, RSI-14, MACD, Bollinger Band position, volume ratio, lagged returns at 1/3/5/10 bars.

**Label:** 1 = "the price was >1% higher 5 days later," 0 = otherwise.

**`XGBoostSignalClassifier.get_feature_importance()`** — Returns features ranked by their contribution to predictions (using the "gain" metric — how much each feature reduces prediction error across all trees).

### Hidden Markov Model — Regime Detector (`models/hmm/`)
**What it does:** Classifies the current market into one of four **regimes**: `low_volatility`, `mean_reverting`, `trending`, or `high_volatility`.

**HMM** (Hidden Markov Model) is a statistical model that assumes the data is generated by a system switching between hidden "states." For markets, these hidden states are regimes we can't observe directly — but we can infer them from volatility, momentum, and yield spread.

**`RegimeDetector.fit(features)`** — Trains a 4-state Gaussian HMM on a (T × N) feature matrix. After fitting, it automatically labels states by ranking on mean volatility.

**`RegimeDetector.predict(features)`** — Returns an array of regime label indices (0-3) for each time step.

**Use case:** A strategy might use a different approach in a "trending" regime vs. a "mean_reverting" regime.

### Transformer (Deferred — `models/transformer/`)
A Transformer-based model is **planned but not yet implemented**. See [`docs/adr/ADR-005-transformer-deferral.md`](../docs/adr/ADR-005-transformer-deferral.md) for the full rationale. In short: the LSTM and XGBoost models cover current needs, and Transformers show their largest advantages on longer sequences and larger datasets than are currently available. The re-engagement criteria include having a GPU training path and at least 30 days of tick-level data.

## Feature Engineering

Both the LSTM and XGBoost models share the same feature computation pipeline in `models/lstm/dataset.py`:

- **Price relatives:** open/high/low expressed as % of close (removes the scale of the absolute price)
- **Returns:** 1-bar, 5-bar, 10-bar percentage returns
- **Moving averages:** SMA-10, SMA-20, SMA-50 (as ratio to current price)
- **RSI-14** (14-day Relative Strength Index)
- **MACD** line, signal line, and histogram
- **Bollinger Band position** (where in the band range is the current price?)
- **Volume ratio** (current volume ÷ 20-day average volume)

XGBoost adds lagged returns (1, 3, 5, 10 bars back) to capture short-term momentum patterns.

## Model Weights Storage

Trained model weights are saved in the `data/ml_weights/` Docker volume:
- LSTM: `data/ml_weights/lstm/{TICKER}.pt` (PyTorch format)
- XGBoost: `data/ml_weights/xgboost/{TICKER}.json` + `{TICKER}_meta.json`
- HMM: `data/ml_weights/hmm/{TICKER}.pkl` (pickle format)

Override the path with the `ML_WEIGHTS_DIR` environment variable.

## How does this connect to the rest of the app?

- `backend/app/tasks/ml_tasks.py` dispatches training jobs and calls into this folder's `train.py` scripts
- `backend/app/api/v1/ml.py` exposes train/predict/feature-importance endpoints
- Feature engineering code in `ml/models/lstm/dataset.py` is shared by both LSTM and XGBoost pipelines
- The `RegimeDetector` from `models/hmm/` can be used by any strategy or analysis endpoint
